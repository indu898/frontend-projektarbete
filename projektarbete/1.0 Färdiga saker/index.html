Illustrera

Float introducerades i Netscape 1.1 år 2002. Det togs fram för att kunna för att flyta text runt en bild. Float i samverkan med div-boxar börjades användas för att skapa sin layout. Innan float använda man tables. När designers insåg möjligheterna med floats började man använda det för att bygga hela sin sidas layout. Det var aldrig ändamålet men det funkade bra under långt tid.

Problemet med floats är att man måste använda sig av extra markup i sin html och extra kod i din css om du vill floata saker efter varandra. En bild som ligger floatad med text kan sticka ut ur sin container. Det är inte en bugg. Det är så det var tänkt. Att folk använde float som layoutverktyg är något som bara hände.

Så tricket är att skapa boxar med fast bredd floata dom som man vill. Om man vill floata mer saker efteråt så läggar man till en 'div class=clear'-box och ger stylar den med en “clearfix”. Man kan också sätta bredden på sina floatade objekt på över 50% vilket tvingar dom att hamna under varandra. Att använda floats som sitt huvudsakliga layoutverktyg skapar onödig kod i markup och css. Float för layout är ett hack.
